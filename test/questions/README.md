## 使用方式

### 按 Flow 使用（推荐）

参考 `test/README.md` Flow 6（RAG 智能问答），按以下顺序执行：

1. **Flow 6.2** → 从 `it-ops-高置信度.md` 选 2-3 题（验证高置信度输出）
2. **Flow 6.3** → 从 `多轮对话测试.md` 选场景 1「VPN 问题深入排查」（验证指代消解）
3. **Flow 6.4** → 从 `低置信度与边界问题.md` 选 2-3 题（验证低置信度引导申告）

### 全面验收

若要进行完整 RAG 质量验收，建议：

- **IT 运维知识库：** 随机抽取 `it-ops-高置信度.md` 中 10 题，置信度通过率应 ≥ 80%
- **信息安全规范库：** 随机抽取 `info-sec-高置信度.md` 中 8 题，置信度通过率应 ≥ 80%
- **低置信度场景：** 执行全部 `低置信度与边界问题.md`，确认低置信度时出现申告引导按钮
- **多轮对话：** 执行至少 2 个场景，验证指代消解准确率 ≥ 70%
- **跨库检索：** 执行 `跨知识库检索测试.md` 第一类（交叉问题）至少 2 题

## 评估标准

| 维度 | 标准 |
|------|------|
| 高置信度 | 置信度 ≥ 60%，回答包含知识库中的具体事实 |
| 低置信度 | 置信度 < 60%，显示「提交申告」引导按钮 |
| 指代消解 | 正确理解"它"、"那个"、"这种"等指代词 |
| 话题切换 | 切换话题后能正确检索对应知识库 |
| 跨库融合 | 同时选择多库时答案融合两个库的信息 |
| 降级处理 | 异常输入不崩溃，有明确提示 |
