企业邮件服务器运维手册
版本：v2.5
最后更新：2026-05-08
维护部门：IT 基础设施组

============================================================
一、邮件系统架构
============================================================

公司邮件系统架构：

Exchange Online (Microsoft 365)：
  - 租户域：company.com
  - 许可证：Microsoft 365 E3 × 500 用户
  - 数据中心区域：东亚（香港）
  - 邮箱容量：默认 50GB + 在线归档 50GB
  - 最大附件：25MB（通过邮件）/ 2GB（通过 OneDrive 链接）

混合部署组件：
  - Azure AD Connect：本地 AD 与 Azure AD 同步（每 30 分钟）
  - Exchange Hybrid Server（本地）：用于管理收件人属性
  - SMTP 中继（本地）：smtp-relay.company.com — 供内部应用/打印机发送通知邮件

============================================================
二、日常运维
============================================================

2.1 账号管理

新员工邮箱创建：
  1. IT 在 AD 中创建域账号
  2. Azure AD Connect 自动同步（30 分钟内）
  3. Microsoft 365 自动分配 E3 许可证 → 邮箱自动创建
  4. 初始密码通过短信发送给员工

离职员工邮箱处理：
  1. IT 禁用 AD 账号 → Azure AD 同步后自动阻止登录
  2. 邮箱转为共享邮箱（保留 30 天）
  3. 设置自动回复：「该员工已离职，请联系 xxx@company.com」
  4. 30 天后管理员删除邮箱（OneDrive 数据同样保留 30 天）

2.2 邮件流监控

每日检查项：
  - Exchange Admin Center → 邮件流 → 邮件追踪
  - 队列中的延迟邮件数量（应 < 10 封）
  - 被隔离的垃圾邮件数量
  - 外发邮件是否有异常激增（可能是账号被盗）

邮件追踪常用 PowerShell：
  # 查询特定发件人的邮件流
  Get-MessageTrace -SenderAddress user@company.com -StartDate (Get-Date).AddHours(-24)

  # 查询被标记为垃圾邮件的邮件
  Get-QuarantineMessage -RecipientAddress user@company.com

三、常见问题处理
============================================================

3.1 用户无法收发邮件

排查步骤：
  1. 确认网络连接（能访问外网）
  2. 确认用户是否被锁定/密码过期（检查 Azure AD 登录日志）
  3. 让用户尝试网页版 https://mail.company.com
     - 网页版正常 → 客户端问题（重建配置文件）
     - 网页版也不正常 → 账号/服务端问题
  4. 检查邮箱是否已满（Exchange Admin → 收件人 → 邮箱使用情况）
  5. 检查是否有未处理的恶意软件检测通知

3.2 邮件被误判为垃圾邮件

处理：
  1. Exchange Admin Center → 审阅 → 隔离
  2. 找到被隔离的邮件 → 释放给收件人
  3. 将发件人域名/地址加入「允许发件人」列表
  4. 向 Microsoft 提交误判报告：https://security.microsoft.com/submissions

3.3 大量用户同时报告邮件延迟

检查项：
  1. Microsoft 365 Service Health Dashboard → 确认是否有服务事件
  2. Exchange Admin Center → 邮件流 → 检查队列
  3. 若 Exchange Online 正常 → 检查本地网络/防火墙
  4. 联系 Microsoft 支持（如有必要）

3.4 钓鱼邮件事件响应

收到钓鱼邮件报告后：
  1. 立即在 Exchange Admin 中搜索该邮件的所有收件人
  2. 从所有未读用户的邮箱中删除该邮件
  3. 将发件域名/URL 加入阻止列表
  4. 通知所有收件人不要点击链接或下载附件
  5. 若有人已点击/已下载 → 启动安全事件响应流程

============================================================
四、邮件安全策略

4.1 反垃圾邮件策略

  连接过滤：Microsoft 默认 IP 阻止列表
  内容过滤：Microsoft 反垃圾邮件引擎 + ATP 安全链接/安全附件
  国际垃圾邮件过滤：阻止来自特定国家/地区的邮件（按需配置）
  批量邮件：标记为「垃圾邮件」的阈值 = 6（默认）

4.2 数据丢失防护 (DLP)

  DLP 策略模板：
    - 中国身份证号码 → 阻止发送并通知管理员
    - 银行卡号（16 位）→ 提示用户 + 记录事件
    - 自定义关键字：「绝密」「商业机密」→ 加密外发邮件

4.3 邮件加密

  外部发送机密信息时：
    - 在邮件主题行添加 [Encrypt] 标签
    - 或使用 Outlook 的「加密」按钮
    - 收件人将收到 OME（Office 365 Message Encryption）加密邮件

五、备份与恢复

邮箱备份：
  - Microsoft 365 原生保留：已删除邮件保留 14 天
  - 诉讼保留（关键岗位）：无限期保留（需合规审批）
  - 第三方备份：Veeam Backup for Microsoft 365（每日备份，保留 1 年）

恢复操作：
  1. 恢复已删除邮件：Exchange Admin → 收件人 → 恢复已删除邮件
  2. 恢复已删除邮箱（30 天内）：Exchange Admin → 收件人 → 已删除邮箱
  3. 从 Veeam 恢复单个邮件/整个邮箱（超出 Microsoft 保留期）

============================================================
六、监控与报告

每日邮件流报告（自动发送至 IT 运维组）：
  - 收件总数 / 发送总数
  - 被阻止的垃圾邮件数量
  - 被隔离的恶意软件数量
  - 入站/出站邮件平均延迟

关键阈值告警：
  - 外发邮件量 > 正常值 × 300%（可能是邮箱被盗）
  - 出站邮件队列 > 100 封
  - ATP 检测到恶意文件 > 5 个/天
