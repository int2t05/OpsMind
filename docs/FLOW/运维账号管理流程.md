# 运维账号管理业务流程

## 1. 业务概述与背景

运维账号管理流程用于支撑 OpsMind 后台管理端的身份认证、角色授权和账号生命周期管理。系统管理员通过后台维护运维人员账号、角色和权限，确保不同岗位只能访问授权范围内的申告处理、知识库维护、配置管理和审计查询功能。

流程覆盖账号创建、登录认证、角色分配、权限加载、账号冻结、账号恢复和敏感操作审计。相关接口定义见 `docs/API/auth/auth.md`、`docs/API/admin/accounts.md`、`docs/API/admin/roles-permissions.md`、`docs/API/admin/audit.md`。

## 2. 适用范围与业务边界

适用范围：

| 范围 | 说明 |
|------|------|
| 后台登录 | 运维人员使用账号密码登录后台 |
| 账号维护 | 管理员创建、修改、冻结、恢复账号 |
| 角色权限 | 管理员维护角色和菜单、按钮、接口权限 |
| 权限加载 | 前端按权限渲染菜单，后端按接口权限鉴权 |
| 审计追踪 | 登录、账号变更、角色权限变更写入日志 |

业务边界：

| 边界项 | 约束 |
|--------|------|
| 账号来源 | MVP 阶段仅管理系统内运维账号，不对接企业账号中心 |
| 鉴权方式 | 后台接口统一使用 Bearer Token |
| 授权模型 | 后端使用角色和权限码控制接口访问 |
| 冻结账号 | 冻结账号不可登录，已有 Token 应在服务端校验中失效 |
| 密码存储 | 数据库仅保存 `password_hash`，不保存明文密码 |

## 3. 参与角色与职责定义

| 角色 | 职责 |
|------|------|
| 系统管理员 | 创建账号、分配角色、冻结恢复账号、维护权限 |
| 运维人员 | 使用账号登录后台并执行授权范围内的业务操作 |
| 后端认证服务 | 校验账号密码、账号状态、签发和刷新 Token |
| 后端授权中间件 | 校验 Bearer Token 和接口权限码 |
| 前端后台应用 | 展示登录页、动态菜单、按钮权限和无权限提示 |
| 审计服务 | 记录登录日志、操作日志和敏感审计日志 |

## 4. 整体业务流程总图说明

```mermaid
flowchart TD
    A[系统管理员进入账号管理] --> B[POST /api/v1/admin/accounts 创建账号]
    B --> C[分配角色]
    C --> D[PATCH /api/v1/admin/roles/{id}/permissions 更新角色权限]
    D --> E[运维人员登录后台]
    E --> F[POST /api/v1/auth/login]
    F --> G{账号密码和状态是否有效}
    G -->|否| H[返回登录失败并写入 sys_login_log]
    G -->|是| I[返回 JWT、用户信息、角色和权限]
    I --> J[GET /api/v1/auth/profile]
    I --> K[GET /api/v1/auth/permissions]
    K --> L[前端动态渲染菜单和按钮]
    L --> M[访问后台业务接口]
    M --> N{后端鉴权是否通过}
    N -->|否| O[返回 401 或 403]
    N -->|是| P[执行业务操作并写入操作日志]
    A --> Q[POST /api/v1/admin/accounts/{id}/freeze]
    Q --> R[账号冻结并写入 audit_log]
    R --> S[冻结账号无法登录]
    A --> T[POST /api/v1/admin/accounts/{id}/restore]
    T --> U[账号恢复并允许重新登录]
```

## 5. 分步详细流程

| 步骤 | 操作 | 责任人/系统 | 输入 | 输出 |
|------|------|-------------|------|------|
| 1 | 查询账号列表 | 系统管理员 | 关键词、角色、状态、分页参数 | `GET /api/v1/admin/accounts` |
| 2 | 创建账号 | 系统管理员 | 用户名、姓名、联系方式、初始密码、角色 ID | `POST /api/v1/admin/accounts` |
| 3 | 写入账号数据 | Go API `server` | 创建账号请求 | `sys_user`、`sys_user_role` |
| 4 | 查询角色列表 | 系统管理员 | 分页参数 | `GET /api/v1/admin/roles` |
| 5 | 查询权限树 | 系统管理员 | 权限类型 | `GET /api/v1/admin/permissions` |
| 6 | 更新角色权限 | 系统管理员 | 角色 ID、权限 ID 列表 | `PATCH /api/v1/admin/roles/{id}/permissions` |
| 7 | 登录后台 | 运维人员 | 用户名、密码 | `POST /api/v1/auth/login` |
| 8 | 校验身份 | 认证服务 | 密码哈希、账号状态、角色关系 | JWT、用户信息、角色、权限 |
| 9 | 加载当前用户 | 前端后台应用 | Bearer Token | `GET /api/v1/auth/profile` |
| 10 | 加载当前权限 | 前端后台应用 | Bearer Token | `GET /api/v1/auth/permissions` |
| 11 | 访问业务接口 | 运维人员 | Bearer Token、业务请求 | 后端授权中间件校验权限码 |
| 12 | 冻结账号 | 系统管理员 | 账号 ID、冻结原因 | `POST /api/v1/admin/accounts/{id}/freeze` |
| 13 | 恢复账号 | 系统管理员 | 账号 ID、恢复原因 | `POST /api/v1/admin/accounts/{id}/restore` |
| 14 | 记录审计 | 审计服务 | 账号变更前后数据、操作者、IP | `audit_log`、`sys_operation_log`、`sys_login_log` |

## 6. 异常分支与兜底处理流程

| 异常场景 | 触发条件 | 处理流程 | 结果 |
|----------|----------|----------|------|
| 用户名重复 | 创建账号时 `username` 已存在 | 返回 `username_exists` 或 `conflict` | 管理员修改用户名 |
| 密码不符合规则 | 初始密码过短或复杂度不足 | 返回 `validation_error` | 管理员重新输入 |
| 登录密码错误 | 密码校验失败 | 返回 `invalid_credentials`，写入失败登录日志 | 拒绝登录 |
| 账号被冻结 | `sys_user.status=2` | 返回 `account_frozen`，写入失败登录日志 | 拒绝登录 |
| Token 过期 | 访问接口时 Token 无效 | 返回 `401 unauthorized` | 前端引导重新登录或刷新令牌 |
| 无接口权限 | 用户角色不包含目标权限码 | 返回 `403 forbidden` | 前端展示无权限提示 |
| 角色权限为空 | 角色未分配菜单或接口权限 | 登录成功但无业务菜单 | 管理员重新配置角色 |
| 冻结当前账号 | 管理员尝试冻结自己 | 返回 `cannot_freeze_self` | 阻止操作 |
| 操作日志写入失败 | 日志服务或数据库异常 | 主业务成功时记录错误日志并告警 | 后续补偿或排查 |

## 7. 准入条件 / 准出条件

准入条件：

| 条件 | 说明 |
|------|------|
| 初始化管理员存在 | `sys_user` 中存在可登录管理员账号 |
| 基础角色存在 | 系统已初始化管理员、运维人员、知识库管理员等角色 |
| 权限树存在 | `sys_permission` 已配置菜单、按钮和接口权限 |
| JWT 配置有效 | Token 签名密钥、过期时间和刷新策略已配置 |
| 审计表可写 | `sys_login_log`、`sys_operation_log`、`audit_log` 可正常写入 |

准出条件：

| 条件 | 说明 |
|------|------|
| 账号创建成功 | `sys_user` 和 `sys_user_role` 写入完成 |
| 登录成功 | 返回 `access_token`、`refresh_token`、用户信息和权限 |
| 权限加载成功 | 前端可按权限展示菜单和按钮 |
| 后端鉴权生效 | 未授权接口返回 `403 forbidden` |
| 冻结生效 | 冻结账号无法再次登录 |
| 审计可追溯 | 敏感操作可通过审计日志查询到操作者、时间、IP 和变更内容 |

## 8. 业务规则、约束与风险点

业务规则：

| 规则 | 说明 |
|------|------|
| 用户名唯一 | `sys_user.username` 必须唯一 |
| 密码不可明文保存 | 后端必须保存密码哈希 |
| 冻结账号不可登录 | 登录时必须校验 `sys_user.status` |
| 后端权限为准 | 前端菜单控制只做体验优化，接口访问以后端鉴权为准 |
| 权限码统一 | 权限码必须与 `docs/API/common/overview.md` 的权限码清单一致 |
| 敏感操作审计 | 账号冻结、恢复、角色权限更新必须写入 `audit_log` |
| 登录日志完整 | 登录成功和失败都必须写入 `sys_login_log` |
| 角色权限去重 | `sys_role_permission` 防止重复授权 |

风险点：

| 风险 | 影响 | 控制措施 |
|------|------|----------|
| 权限配置过大 | 普通运维人员越权访问配置和审计功能 | 最小权限原则、角色评审 |
| Token 泄露 | 非授权人员访问后台接口 | 短期 Token、刷新令牌、服务端状态校验 |
| 冻结后 Token 仍可用 | 被冻结人员继续操作系统 | 鉴权中间件实时校验账号状态 |
| 密码弱口令 | 后台被暴力破解 | 密码复杂度、登录失败限流、审计告警 |
| 审计缺失 | 无法追踪敏感操作 | 账号和权限变更强制写审计日志 |
| 初始化权限错误 | 管理员无法进入后台 | 提供初始化脚本和默认管理员恢复流程 |
