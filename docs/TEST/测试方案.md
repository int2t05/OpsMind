# 运维数字员工系统测试方案

| 项目 | 内容 |
| --- | --- |
| 文档版本 | v1.0 |
| 生成日期 | 2026-05-15 |
| 测试对象 | OpsMind 运维数字员工系统 MVP |
| 测试依据 | `docs/PRD.md`、`docs/TECH.md`、`docs/DB.md`、`docs/API/`、`docs/FLOW/`、`docs/PLAN.md` |
| 测试范围 | 后端接口、前端页面、数据库迁移、RAG/模型适配、文件上传、权限、审计、部署和端到端业务闭环 |

## 1. 测试目标

本测试方案用于验证 OpsMind MVP 是否满足智能问答、申告处理、知识库管理、账号权限、日志审计、系统配置、基础看板和 Docker Compose 部署要求。

测试目标：

| 目标 | 验证重点 |
|------|----------|
| 功能正确 | 核心业务流程可按 PRD 和 FLOW 文档完成 |
| 接口一致 | 实际接口路径、请求、响应、错误码与 `docs/API` 和 `openapi.yaml` 一致 |
| 数据可靠 | 数据库迁移、状态流转、事务、审计记录符合 `docs/DB.md` |
| 权限安全 | 后台接口必须认证和授权，匿名接口必须限流 |
| AI 可降级 | AnythingLLM、vLLM 不可用时能返回兜底并引导人工申告 |
| 部署可复现 | 按 README 和部署脚本可启动并演示完整闭环 |

## 6. 缺陷等级定义

| 等级 | 定义 | 示例 | 处理要求 |
|------|------|------|----------|
| P0 阻塞 | 核心流程不可用或数据损坏 | 无法登录、数据库迁移失败、申告创建失败 | 当日修复，阻塞发布 |
| P1 严重 | 核心功能异常但有临时绕过方式 | 知识发布失败、权限绕过、附件无法下载 | 1-2 天内修复 |
| P2 一般 | 非核心功能异常或局部体验问题 | 筛选条件不准确、提示文案错误 | 当前迭代修复 |
| P3 轻微 | 不影响使用的展示或文档问题 | 表格列宽、空状态文案不统一 | 视排期修复 |

## 8. 缺陷排查技巧

| 场景 | 排查顺序 |
|------|----------|
| 接口返回 500 | 查看请求 ID -> 后端日志 -> 数据库错误 -> 外部服务调用日志 |
| 接口返回 401 | 检查 Authorization Header -> Token 是否过期 -> 账号是否冻结 |
| 接口返回 403 | 检查用户角色 -> 权限码清单 -> Casbin policy -> 路由权限配置 |
| 问答失败 | 检查 AnythingLLM 地址 -> vLLM 地址 -> 知识库编码 -> 后端 RAG 适配日志 |
| 文件上传失败 | 检查文件大小类型 -> MinIO 连接 -> bucket 权限 -> `sys_file` 记录 |
| 知识同步失败 | 检查 NATS 事件 -> Worker 日志 -> AnythingLLM 响应 -> pgvector 维度 |
| 前端页面空白 | 检查浏览器控制台 -> 路由守卫 -> API 代理 -> 权限菜单数据 |
| 数据不一致 | 检查事务边界 -> repository 日志 -> 数据库约束 -> 审计记录 |

## 9. 测试风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 外部 AI 服务不稳定 | 问答和知识同步无法稳定测试 | 准备 mock RagClient 和 mock ModelClient |
| 本地资源不足 | vLLM 启动慢或无法启动 | 使用远程 OpenAI-compatible 服务或固定 mock 响应 |
| 测试数据污染 | 状态流转和统计结果不稳定 | 每轮测试前执行 `scripts/seed.sql` 重置数据 |
| 权限数据复杂 | 用例执行结果不确定 | 固定三类角色和权限矩阵 |
| E2E 不稳定 | 自动化结果误报 | 使用明确等待条件，保留 trace、截图和视频 |
| 文档与实现漂移 | 接口测试失败难定位 | 每个里程碑执行 API 文档对齐检查 |
